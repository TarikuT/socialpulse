# QA workflow

This folder holds CSVs used to measure how well SocialPulse's GPT-4o-mini classifier matches a human native-speaker's sentiment judgments. **Nothing here is committed to git** — comments and labels stay on your local machine.

## To run a labeling pass

1. On the deployed app, run an analysis on a video. Download the CSV.
2. Save it to this folder. Suggested naming: `<short_video_name>.csv` (e.g., `rophnan_album.csv`).
3. From the `socialpulse/` directory, run:
   ```bash
   streamlit run app/qa_labeler.py
   ```
4. Pick the CSV from the dropdown. Label comments one at a time. Labels save after every click.
5. When you're done, the page shows overall agreement, per-class recall, Cohen's kappa, the confusion matrix, and the specific disagreements.

## Files

- `<name>.csv` — raw SocialPulse output, the data you're labeling.
- `labels_<name>.csv` — your human labels alongside GPT's, generated by the labeler.

## Targets

- **≥85% agreement, kappa ≥0.7:** publish public analyses with confidence.
- **70–85%:** inspect the confusion matrix, tune the system prompt, re-run.
- **<70%:** harder fix — consider a different model or restricting language scope.
